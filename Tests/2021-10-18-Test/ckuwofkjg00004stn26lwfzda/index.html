# 心路历程：

这场属于是输麻了，开考 2min 猜到 T1 结论，写了个暴力拍一拍，然后由于暴力ckmin时取模了\color{red}\texttt{暴力ckmin 时取模了}暴力ckmin 时取模了。我随便拍都比我的答案小，然后直接认为这个结论假了。。。。

大概 30min 时机房同学萌纷纷通过了此题，心态爆炸，然后通过各种途径反复拆 T1 柿子，都证明了我猜的结论是对的，但我总坚信是我推错了… 然后就十分自闭，直到 1h20min 时，我实在找不出我的证明哪里锅了，然后把我的 hack 数据交给 murmur 一看，草我咋对拍写挂了。。。

然后随便写了个并茶几找了下次数在 1h30min 交了，还写了拍跟别人去拍，一来二去就到了 2h ，我才开始开 T2

T2 是一个比较简单的模拟（？） 但我感觉自己向来不会做这种二维的题，开始写了个代码和 ysh 拍挂了，然后我又写了棵 SGT 发现还有地方没处理，然后又匆匆删了 SGT 冲另个一次函数找交点的做法，此时只有 10min 下考了，赶紧冲了个 T4 暴力，冲对了。

感觉到自己对有点细节的题目处理不是很到位，加上心态爆炸以及对概率长期以往的惧怕等因素导致 T3 都没想。实质上 T3 正解并不难想。

# Solutions

# T1

两个数组 a,ba,ba,b ，可任意交换 aaa 中元素，使得：

∑i=1n(ai−bi)2\sum\limits_{i=1}^{n}(a_i-b_i)^2i=1∑n​(ai​−bi​)2

最小，并输出最小操作次数。

大胆猜测把两个数组从大到小排序直接来就是答案，操作次数看做两个排列，使用并查集可以解决。

证明考虑若存在逆序对交换会使得答案变小。

# T2

n×nn\times nn×n 的棋盘，mmm 个主教，求不能被吃的位置数。

每个主教相当于一条斜率为 111 和 −1-1−1 的直线，模拟这个过程用线段树找交点即可在线做。

但离线做可能会更为简单？

# T3

nnn 头牛，每头牛有 pip_ipi​ 的概率直接患病，有 qiq_iqi​ 的概率感染第 toito_itoi​ 头牛，求每头牛被感染的概率。

每个点一条出边不难想到这是个内向基环树森林，树的部分严格有序直接 Dp 就好，环的部分由于每个点出度为 111 ，一定是一个接一个的环，拆两半复制一份对于每个点，他前面 nnn 条边如何转移到它？那就枚举它前面第一条断掉的边，然后中间那段任意一头牛患病即可使他患病，还有前面没断边的情况。这个使用一次加一棵数字考虑与前面的差异可以 O(1)O(1)O(1) 转移。

# T4

一张有向图，计数保留若干条边后存在环的方案。

考虑一步容斥，问题变成统计 DAG 数

然后我懂但不完全懂
