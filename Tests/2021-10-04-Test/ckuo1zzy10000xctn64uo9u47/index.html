# 常数国

nnn 张牌围成一个环，每张牌有个初始权值 aia_iai​ ，有 qqq 次询问，每次询问一个区间一个数字，表示用一张大小为给定的牌到区间里扫一遍，若区间内牌面大于手中的牌则交换，问最后的手牌是啥。

n≤4×105n\le 4\times 10^5n≤4×105

this

考虑分块来做这样一个问题。

对于整块，一张牌进出它后出来的一定是这个整块当前的最大值。所以在整块中我们并不关注每个位置具体有什么数字，我们只关注整体的状况，所以只需用个大根堆维护即可。

对于散块，我们需要考虑每个位置是啥了，但散块不大于是可以暴力修改，但这个块整体可能被作为整块经过过若干次，而作为整块时我们并不关注其具体值，所以在暴力修改前我们得重构一下。

考虑怎么重构，每次一张牌经过一个整块时我们在这个整块打上一个标记，考虑一个整块有若干个 tag 时最终是怎么样的。我们发现，一定是对于每个位置，找到当前 tag 里最小的那个看看能不能与其交换，若能就放上去然后把当前位置的数字丢进 tag 里。注意这里我们运用了把 tag 丢一起处理考虑最终情况的思想，我觉得若没有类似经验还是难以在考试想到的，所以积累一下。这个重构的过程我们可以用小根堆来模拟。

但这样常数过不去，容易发现我们并不需要时时维护小根堆，只需在重构的时候使用即可，于是我们使用 stl 中的几个函数

make_heap(a.begin(), a.end(), cmp) 表示按 cmp 比较大小方式建了一个大根堆，但是一个数组的形式，第一个位置为根。 x×2,x×2+1x\times 2,x\times 2 + 1x×2,x×2+1 为 xxx 的左右儿子。

pop_heap(a.begin(), a.end(), cmp) 表示把根与最后一个元素交换，然后对前 n−1n-1n−1 个元素做了一遍 make_heap

push_heap(a.begin(), a.end(), cmp) 表示插入一个元素。

注意记得对于重构时的 tag ，我们一次重构后要清空，否则会 T

# 世界线修理

两棵树，给每个点赋值，要求每棵树中每个点的子树权值和绝对值为 111

n≤105n\le 10^5n≤105

This

我们大胆猜测，可以用 1,−1,01,-1,01,−1,0 来构造出解

可以发现，若两棵树每个点的儿子的奇偶性不同一定无解。

然后，对于一个点，他的每个儿子的权值和绝对值为 111 ，相当于每个点面临同样一个子问题。

若他有奇数个儿子，则我们考虑直接给他赋值为 000 ，我们需要保证权值和为 111 的和权值和为 −1-1−1 的儿子数刚好相差 111

若他有偶数个儿子，则我们考虑给他赋值为 1/−11/-11/−1 ，保证权值和为正负一的儿子相同。

在考试时，我是直接钦定前一半儿子为正后一半为负，当前点看祖先对其的要求，但这有一个大问题，就是一个点在第一棵树中可能是前一半儿子，但在第二棵树中是后一半儿子，而我的随机数 gen 连这都查不出… 然后 FST 了

这个锅修一修就可以 A 了，我们看看标算的做法，在每个点儿子处让子树内权值为奇数两两匹配（一个为 111 ，一个为 −1-1−1）然后跑黑白染色。为什么可以直接跑黑白染色，因为它无奇环，是个二分图。
