# Solutions:

# tournament

2n2^n2n 个人比赛，每人有个独一无二的能力值 ( 1→2n1\to 2^n1→2n 的排列)，两两比赛胜者继续，求把一个人交换至多一次对手后能赢的最多次数。

n≤18n\le 18n≤18

简单贪心，但我没想出来

大概是每一层有个最大值，可以被你交换消掉，于是你能赢的最大次数大概就是从高层往低层看你第一个能偏序的就行了。

# inversion

树上统计长度为 kkk 的路径逆序对数。

容易发现 1h 后发现，逆序对正着来一遍反着来一遍，就是统计树上长度为 kkk 的路径树乘个 k×k+12k\times k+1\over 22k×k+1​

# tree

一棵基环树，统计删除一条边后直径长度。

大力分讨

O(nlog⁡n)O(n\log n)O(nlogn) 永不为奴

给一个 010101 串，kkk 轮次，每轮看到一个 010101 就交换，问最后的结果.

大概是，你考虑 000 的旅途，大概都是往右走的，而两个 000 的相对位置显然不会改变，于是你从右往左对每个 000 分别考虑，若他左边是 111 是可以一直往右走，你把 t−xt-xt−x 图像画出来，斜率是 +1+1+1 的。

而若左边有其它 000 ，你可能出现的情况是走着不能走了，因为你右边是个 000 ，必须等它先走完你才能走，这是一个排队的过程，会在 t−xt-xt−x 图像上形成一条条竖线。

你考虑你已经知道了一个点的 t−xt-xt−x 折线后对更左边点的影响，你把这条折线向左平移一位再向上平移一位，这是一条 “限制折线”，代表你左边点，每一时刻最多能到哪里，那你左边点的折线就是，从它起点出发一条斜率为 111 的斜线，第一次交这条限制直线后就一直沿着它走。

考虑啥时候会出现竖线，一定是，若干个 000 合在一起的时候，那竖线条数是 nnn 的级别的，可以均摊。

这个代码十分玄妙，有 slope trick 那味了

那个队列，维护的是竖线的纵坐标。
